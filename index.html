What is Mixlist?

Mixlist is a collaborative music application designed to generate playlists that reflect the combined musical tastes of a group.

While music streaming services provide personalized recommendations for individuals, they offer limited tools for creating playlists that adapt to the preferences of multiple listeners in shared environments such as social gatherings, workplaces, or small businesses.

Mixlist addresses this challenge by allowing users to authenticate through Spotify, join a shared session, and contribute their listening preferences to a dynamically generated group playlist. The system aggregates listener taste profiles and assigns weighted values to songs in order to construct playlists that balance the preferences of all participants.

By leveraging Spotify’s API and real-time session management, Mixlist creates a shared listening experience that adapts as participants join or leave the session.

Tech Stack

    Frontend:
React JS, Tailwind CSS : React's design allows Mixlist's UI to update automatically as session data changes. Tailwind CSS provides a consistent user experience across the platform.
    Backend:
Web server (Node.js + Express JS): - Mixlist uses Node.js with Express to handle API requests, manage sessions, and communicate with the Spotify API and OAuth services. This architecture allows multiple users to participate in a session while keeping API usage efficient and within Spotify’s rate limits.
    Application Logic:
Web server (Node.js): - Node.js processes listener taste profiles and combines them into a shared group preference. This enables Mixlist to dynamically generate and update playlists as users join or leave a session.
    Database:
MySQLite - Mixlist uses SQLite to store lightweight session metadata such as session IDs, listener references, and playlist identifiers. Music files and personal user data are not stored locally, as playlists are created and managed through the Spotify API.

Ethics & Governance

IRB/CITI Status:

Music preferences can be very personal and may sometimes reveal sensitive information about a person. For example, certain genres, artists, or podcasts might reflect someone’s beliefs, emotional state, or interests. Because Mixlist uses Spotify listening data to generate a collaborative playlist, it is important that users understand how their information is being used during a session.

To address this, users must log in and authorize access through Spotify OAuth before they can join a Mixlist session. This authorization allows the system to temporarily access the user’s music preferences in order to generate the shared playlist. Only the data necessary to create the playlist is used, and the application does not access unrelated personal information.

When creating a collaborative playlist from multiple participants, recommendation systems can sometimes favor more popular songs or genres. This could result in certain users’ preferences being underrepresented if their music taste is less common.

Mixlist attempts to reduce this issue by combining the music preferences of all participants and assigning weights to tracks based on the group’s overall listening preferences. By aggregating taste profiles from each participant, the system aims to represent everyone in the session rather than prioritizing only the most popular songs.

Another important ethical consideration is how long participant data should remain accessible. Users may not expect their listening data to remain available after the collaborative session ends.

To protect user privacy, Mixlist stores listening data and Spotify access tokens only for the duration of an active session. If a listener leaves the session or the host ends the Mixlist, the associated listener data and tokens are removed from the system. This approach ensures that participant data is only used temporarily for the purpose of generating the playlist.
